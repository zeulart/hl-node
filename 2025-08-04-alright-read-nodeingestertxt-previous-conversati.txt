> why did you map this volume in the compose ?       - hl-data:/home/hluser/hl/data:ro
  ⎿  ⧉ Selected 1 lines from docker-compose.yml in Visual Studio Code

⏺ The :ro (read-only) flag was mapped for the log-streamer service to ensure:

  Security & Safety:
  - Prevents the streaming service from accidentally modifying node data
  - Read-only access is all that's needed for log tailing
  - Protects against bugs that could corrupt blockchain data

  Resource Isolation:
  - Ensures the streamer can't interfere with node operations
  - Node has exclusive write access to its data directory
  - Clear separation of concerns between data producer (node) and consumer (streamer)

  Best Practice:
  - Follows principle of least privilege
  - The streamer only needs to read log files, never write
  - Reduces risk if the streaming container is compromised

  The node writes to /home/hluser/hl/data while the log-streamer only reads from it,
  making :ro the appropriate permission level.

  log-streamer:

  docker-compose up redis log-streamer

  - redis - Required dependency for the log-streamer

  # Run in detached mode
  docker-compose up -d redis log-streamer

  # Run only log-streamer (Redis will start automatically due to depends_on)
  docker-compose up log-streamer

  # Build and run
  docker-compose up --build log-streamer

  Note: The log-streamer needs the hl-data volume to exist (created by the node
  service), so if you've never run the full stack, you might need to create the volume
  first or run the node service once to populate the log files.
